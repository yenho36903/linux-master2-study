# 1과목 리눅스 운영 및 관리
1.파일 시스템 관련 명령어 
1.1 권한 및 그룹 설정
모든 파일과 디렉터리에는 접근권한,소유권가 부여됨 
소유권 
파일 혹은 디렉터리에 대한 사용자와 그룹들의 소유 권한
관련 명령어
 chown [옵션] 소유자[:그룹명] 파일명 
 사용자 소유권,그룹 소유권 변경
 chgrp [옵션] 그룹명 파일명 
 그룹 소유권 변경 
 옵션
 -R : 하위 디렉터리를 포함 전체 변
허가권 
ls -l -> 파일유형과 허가권 알수있음
- 파일사용자 : 소유자,그룹,기타 사용자
- 권한: 읽기(r),쓰기(w),실행(x)
- 명령어
 chmod (파일,디렉터리의 접근 허가권 변경)
 옵션 : - R : 하위 디렉터리를 포함해서 모든 디렉터리 및 파일의 권한 변경
 숫자로 바꾸기 : r은 4 , w 은 2 , x은 1
 ex: rw- : 42- , r-- : 4--, rwx : 421
기호로 바꾸기 :
사용자 (u : 소유자, g : 그룹 , o : 기타사용자 , a : 모든 사용자 u+g+o
연산자 (+ : 허가권 부여, - : 허가권 제거 , = : 특정 사용자에게 허가권 지정 )
권한 (읽기(r),쓰기(w),실행(x))
umask (기본 허가권)
- S : 문자로 표기
파일 기본권한 : 666 , 디렉터리의 기본권한 : 777
-특수 권한
해당 프로세스의 root 권한을 임시로 가져오는 기능
SetUID : 파일 소유자와 같은 권한
SetGID: 파일의 그룹과 같은 권한
Sticky bit : 공용 디렉터리를 사용할때
- 디스크 쿼터
파일 사스템마다  사용자나 그룹 생성 할수있는 파일 용량/ 개수 제한 , 단위의 용량 제한과 inode의 개수를 제한
관련 명령어
edquota : 편집기를 이용해 사용자나 그룹에 디스크 사용량을 할당
setquota : 편집기 사용하지 않고 사용자가 직접 사용자나 그룹에 디스크 사용량을할당
xfs_quota : Centos 7에서 사용자의 디스크 사용량 제한 (-x-c옵션)
requota : 파일 시스템의 사용자 또는 그룹에 대한 디스크 사용량 보고서를 생성하고  보고
1.2 파일 시스템의 관리
-리눅스 전용 디스크 기반 파일 시스템
ext(ext1) - 리눅스 초기에 사용되던 파일시스템, 호환성이 없음
ext2 - ext파일 시스템의 다음 버전, 고용량 디스크 사용을 엽두에 두고 설계
ext3 - 저널링 , ACL을 통한 접근 제어 지원
ext4 - ext2와 ext3 호환성이 있는 확장 버전
-저널링 파일 시스템
시스템의 비정상적인 종료시 저널(로그)을 이용해 빠르면서도 안정적인 복구가 가능
ext3, ext4, XFS, JFS, ResierFS
네트워크 파일 시스템
- SMB : 삼바 파일 시스템을 마운트 지정
- CIFS : SMB를 확장한 파일 시스템
- NFS : 네트워크 파일 시스템, 파일 공유 및 파일 서버로 사용됨
파일 관련 명령어
- mount : 특정 디바이스를 특정 디렉터리처럼 사용하기위해 장치와 디렉터를 연결 (-t: 파일 시스템 , - o : 추가 설정)
- umount : 마운트 해제
- eject : 이동식 보조기억장치등과 같은 미디어를 해제하고 장치를 제거
- fdisk : 파티션 생성,삭제,결정 (-l: 파티션 목록, -v :버전 정보 표시, d : 삭제)
